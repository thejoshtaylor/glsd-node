<objective>
Wire PersistenceManager into the GSD callback path so that keyboard-triggered sessions are persisted for /resume. Currently enqueueGsdCommand starts workers without OnQueryComplete, so GSD-only sessions are never saved.

Purpose: PERS-01 requires session state to be saved to JSON. The text handler path already does this, but the GSD callback path does not. This plan threads the existing `persist` param (already on HandleCallback) down to enqueueGsdCommand and sets OnQueryComplete matching the text.go pattern.

Output: enqueueGsdCommand sets OnQueryComplete on WorkerConfig; all intermediate functions thread persist param; GSD-triggered sessions appear in /resume.
</objective>
