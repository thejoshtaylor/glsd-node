# Windows Service Installation (NSSM)

The bot runs as a Windows Service via NSSM (Non-Sucking Service Manager). NSSM wraps the Go binary and handles automatic start at boot, log file redirection, environment variable injection, and service restart on failure.

## Prerequisites

1. Build the bot binary: `go build -o claude-telegram-bot.exe .`
2. Download NSSM from https://nssm.cc/download (2.24 stable). Extract `nssm.exe` to a permanent location (e.g., `C:\tools\nssm.exe`).
3. Locate your tool paths: Claude CLI (typically `C:\Users\<user>\AppData\Roaming\npm\claude.cmd`), pdftotext (optional, from Poppler for Windows).

## Install Service

Run all commands in an Administrator Command Prompt.

1. Install: `nssm install ClaudeTelegramBot "C:\path\to\claude-telegram-bot.exe"`
2. Set working directory: `nssm set ClaudeTelegramBot AppDirectory "C:\path\to\bot"`
3. Set environment variables via `nssm set ClaudeTelegramBot AppEnvironmentExtra` with all required vars: TELEGRAM_BOT_TOKEN, TELEGRAM_ALLOWED_USERS, CLAUDE_CLI_PATH, PDFTOTEXT_PATH, OPENAI_API_KEY, ALLOWED_PATHS, DATA_DIR
   - Use `AppEnvironmentExtra` (adds to inherited env), NOT `AppEnvironment` (replaces entire env)
   - For paths with spaces, use the NSSM GUI: `nssm edit ClaudeTelegramBot`
4. Configure log files: `nssm set ClaudeTelegramBot AppStdout "C:\logs\claude-bot.log"` and `AppStderr`, with `AppStdoutCreationDisposition 4` for append mode
5. Configure restart: `nssm set ClaudeTelegramBot AppRestartDelay 5000`
6. Start: `nssm start ClaudeTelegramBot`

## Manage Service

- Status: `nssm status ClaudeTelegramBot`
- Stop: `nssm stop ClaudeTelegramBot`
- Restart: `nssm restart ClaudeTelegramBot`
- Edit: `nssm edit ClaudeTelegramBot`
- Remove: `nssm stop ClaudeTelegramBot && nssm remove ClaudeTelegramBot confirm`

## Environment Variables Reference

Full table of all env vars (TELEGRAM_BOT_TOKEN, TELEGRAM_ALLOWED_USERS required; CLAUDE_CLI_PATH, CLAUDE_WORKING_DIR, ALLOWED_PATHS, OPENAI_API_KEY, PDFTOTEXT_PATH, DATA_DIR, AUDIT_LOG_PATH, RATE_LIMIT_ENABLED, RATE_LIMIT_REQUESTS, RATE_LIMIT_WINDOW optional with documented defaults).

## Troubleshooting

Sections for: Service won't start (check error log, verify env vars, test binary manually), Claude CLI not found (set CLAUDE_CLI_PATH explicitly), Temp directory issues (NSSM runs as SYSTEM by default, configure ObjectName for user account), PDF extraction not working (verify pdftotext installed, set PDFTOTEXT_PATH).
